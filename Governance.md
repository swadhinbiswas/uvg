# Governance

**Date:** 2026-06-04
**Status:** APPROVED

---

## Project Structure

GVX is an open-source project governed by a meritocratic model.

---

## Roles

### Contributor

Anyone who contributes to GVX.

**Requirements:**
- Submit a PR that is merged

**Rights:**
- Participate in discussions
- Submit PRs
- Report issues

### Committer

A contributor with write access to the repository.

**Requirements:**
- 10+ merged PRs
- Sponsorship from 2 maintainers
- Maintainer vote (majority)

**Rights:**
- Merge PRs
- Review code
- Commit to main
- Participate in maintainer votes

### Maintainer

A committer with decision-making authority.

**Requirements:**
- 50+ merged PRs
- 6+ months of active contribution
- Sponsorship from 3 maintainers
- Maintainer vote (2/3 majority)

**Rights:**
- All committer rights
- Decide feature direction
- Approve releases
- Manage repository settings
- Vote on governance changes

### Lead Maintainer

The primary decision-maker for GVX.

**Requirements:**
- Maintainer for 12+ months
- Maintainer vote (2/3 majority)

**Rights:**
- All maintainer rights
- Tie-breaking vote
- Release approval authority
- External representation

---

## Decision Making

### Consensus

Most decisions are made by consensus in discussions.

### Voting

If consensus cannot be reached:

| Decision Type | Threshold |
|---------------|-----------|
| Feature direction | Maintainer majority |
| Breaking change | 2/3 maintainer |
| Governance change | 2/3 maintainer |
| Release approval | Lead maintainer |
| Role promotion | 2/3 maintainer |
| Role removal | 3/4 maintainer |

### Veto

The lead maintainer may veto any decision. The veto can be overridden by a 3/4 maintainer vote.

## Maintainer Responsibilities

### Technical

- Review PRs
- Guide architecture
- Ensure quality
- Maintain security
- Monitor performance

### Community

- Welcome new contributors
- Resolve conflicts
- Enforce code of conduct
- Represent GVX externally

### Operational

- Manage releases
- Manage infrastructure
- Manage CI/CD
- Manage documentation

---

## Maintainer Meetings

### Frequency

Monthly maintainer meeting.

### Agenda

- Review open issues
- Review open PRs
- Discuss upcoming releases
- Discuss community feedback
- Discuss architecture decisions
- Discuss governance issues

### Quorum

50% of maintainers for a quorum.

### Minutes

Meeting minutes are published in the repository.

---

## Conflict Resolution

### Level 1: Discussion

Parties discuss the issue in private.

### Level 2: Mediation

A neutral maintainer mediates the discussion.

### Level 3: Vote

Maintainers vote on the issue.

### Level 4: Lead Maintainer

The lead maintainer makes the final decision.

---

## Role Removal

### Grounds for Removal

- Inactive for 12+ months
- Code of conduct violation
- Security violation
- Repeated poor judgment

### Process

1. Issue raised by any maintainer
2. Discussion among maintainers
3. Vote (3/4 majority required)
4. If passed, role is removed
5. Removed person may appeal after 6 months
