UVG computes a **deterministic SHA-256 fingerprint** of each runtime's state. This fingerprint enables instant runtime reuse and verifies reproducibility.

## What Is Fingerprinted?

The fingerprint input includes:

```python
FingerprintInput = {
    "packages": [
        ("name", "version", "hash"),  # Sorted alphabetically
        ...
    ],
    "python_version": "3.12",
    "platform": "linux",
    "architecture": "x86_64",
}
```

### Key Properties

- **Sorted packages** — Order doesn't affect fingerprint
- **Include hash** — Same version, different wheel = different fingerprint
- **Include Python version** — Same deps, different Python = different fingerprint
- **Include platform** — Same deps, different OS = different fingerprint

## Fingerprint Computation

```python
import hashlib
import json

def compute_fingerprint(input: FingerprintInput) -> str:
    data = json.dumps(input, sort_keys=True, separators=(',', ':'))
    hash_bytes = hashlib.sha256(data.encode('utf-8')).digest()
    return f"runtime_{hash_bytes[:4].hex()}"
```

The result is a string like `runtime_a1b2c3d4`.

## Determinism Guarantee

The same input **always** produces the same fingerprint:

```python
# Run 1
compute_fingerprint(input)  # "runtime_a1b2c3d4"

# Run 2 (different machine, same input)
compute_fingerprint(input)  # "runtime_a1b2c3d4"

# Run 3 (same machine, different time)
compute_fingerprint(input)  # "runtime_a1b2c3d4"
```

## Runtime Reuse

When you run `uvg sync`, UVG:

1. Computes the fingerprint for the current dependency set
2. Checks if a runtime with that fingerprint already exists
3. If yes, **reuses it instantly** (no reconstruction needed)
4. If no, builds a new runtime

```bash
# First sync: build from scratch
uvg sync
# Fingerprint: runtime_a1b2c3d4
# Time: 1.2s

# Second sync: fingerprint match, reuse
uvg sync
# Fingerprint: runtime_a1b2c3d4 (same!)
# Time: 0.05s
```

## When Fingerprints Change

The fingerprint changes when:

| Change | Fingerprint Changes? |
|--------|---------------------|
| Add a package | Yes |
| Remove a package | Yes |
| Update a package version | Yes |
| Change Python version | Yes |
| Change platform | Yes |
| Same package, different wheel | Yes |
| Re-run sync (no changes) | No |

## Fingerprint Format

```
runtime_<8 hex characters>
```

Example: `runtime_a1b2c3d4`

The 8-character prefix provides 2^32 possible fingerprints, which is sufficient for practical use while remaining human-readable.

## Manifest Storage

The fingerprint is stored in the runtime manifest:

```json
{
  "fingerprint": "runtime_a1b2c3d4",
  "python_version": "3.12",
  "created_at": "2024-01-15T10:30:00Z",
  "packages": [...]
}
```

## Verification

Verify that a runtime matches its fingerprint:

```bash
uvg verify --runtime
```

This recomputes the fingerprint from the current runtime state and compares it to the stored fingerprint. If they don't match, the runtime has been modified.

## Performance

Fingerprint computation is fast:

| Packages | Time |
|----------|------|
| 10 | 0.008 ms |
| 100 | 0.012 ms |
| 1000 | 0.025 ms |
