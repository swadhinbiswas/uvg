Thank you for your interest in contributing to UVG!

## Development Setup

### 1. Clone the Repository

```bash
git clone https://github.com/swadhin/uvg.git
cd uvg
```

### 2. Install Dependencies

```bash
uv sync
```

### 3. Run Tests

```bash
uv run pytest
```

### 4. Run Linting

```bash
uv run ruff check .
uv run ruff format --check .
uv run mypy src/
```

## Coding Standards

### Python Style

- Follow [PEP 8](https://peps.python.org/pep-0008/)
- Use `ruff` for linting and formatting
- Use `mypy` for type checking
- Maximum line length: 100 characters

### Type Hints

All code must be fully type-hinted:

```python
def compute_fingerprint(packages: list[PackageIdentity]) -> str:
    ...
```

### Docstrings

Use Google-style docstrings:

```python
def sync(project: Project) -> Runtime:
    """Build or update the project runtime.

    Args:
        project: The project to sync.

    Returns:
        The constructed runtime.

    Raises:
        RuntimeError: If sync fails.
    """
```

### Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add workspace sync command
fix: resolve symlink path correctly
docs: update installation guide
test: add fingerprint benchmark
refactor: extract store index logic
```

## Testing

### Run All Tests

```bash
uv run pytest
```

### Run Specific Tests

```bash
uv run pytest tests/unit/test_store.py
uv run pytest tests/unit/test_store.py::TestStore::test_create_object
```

### Run Benchmarks

```bash
uv run pytest tests/benchmarks/ -v
```

### Coverage

```bash
uv run pytest --cov=src/uvg --cov-report=html
```

## Pull Request Process

### 1. Create a Branch

```bash
git checkout -b feature/my-feature
```

### 2. Make Changes

- Write code following the standards above
- Add tests for new functionality
- Update documentation

### 3. Run Checks

```bash
uv run pytest
uv run ruff check .
uv run ruff format .
uv run mypy src/
```

### 4. Commit

```bash
git add .
git commit -m "feat: add my feature"
```

### 5. Push and Create PR

```bash
git push origin feature/my-feature
```

## Documentation

### Adding Documentation

1. Create a new `.mdx` file in `docs/src/content/docs/`
2. Add frontmatter with `title` and `description`
3. Update the sidebar in `docs/astro.config.mjs`

### Documentation Style

- Use clear, concise language
- Include code examples
- Link to related pages
- Use callouts for notes, tips, and warnings

## Reporting Issues

### Bug Reports

Include:
- UVG version
- Python version
- Platform
- Steps to reproduce
- Expected behavior
- Actual behavior

### Feature Requests

Include:
- Problem description
- Proposed solution
- Use case
- Alternatives considered

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Follow the project's governance model
