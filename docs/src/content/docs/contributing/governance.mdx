UVG follows a transparent, community-driven governance model.

## Roles

### Contributors

Anyone who contributes to UVG through code, documentation, or discussion.

### Maintainers

Trusted contributors with write access to the repository. Responsibilities:
- Review and merge pull requests
- Triage issues
- Guide project direction
- Mentor new contributors

### Lead Maintainer

Coordinates maintainer activities and has final say on disputed decisions.

## Decision Making

### Consensus

Most decisions are made through consensus in discussions.

### Voting

For disputed decisions, maintainers vote. Simple majority wins.

### Lead Maintainer Tiebreaker

If voting results in a tie, the lead maintainer decides.

## Contribution Process

1. **Proposal** — Open an issue or discussion
2. **Discussion** — Community provides feedback
3. **Implementation** — Contributor creates a PR
4. **Review** — Maintainers review the PR
5. **Merge** — PR is merged after approval

## Release Process

### Versioning

UVG follows [Semantic Versioning](https://semver.org/):

- **MAJOR** — Breaking changes
- **MINOR** — New features (backwards compatible)
- **PATCH** — Bug fixes

### Release Steps

1. Update version in `pyproject.toml`
2. Update changelog
3. Create release tag
4. Build and publish
5. Announce release

## Communication

- **GitHub Issues** — Bug reports and feature requests
- **GitHub Discussions** — General discussion
- **Pull Requests** — Code review

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Follow professional communication standards
